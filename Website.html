When should you use Git for a project?

You should use Git for a project when you want to have backup versions available or want to collaborate with others.


2

What kind of files/info should be saved in a Git repository?

Non-confidential files/info should be saved in a Git repository, such as code or text files you don't mind sharing with the world.


3

What types of files/info should not be included in a Git repository?

Private files/info should not be included in a Git repository, such as personal information or passwords.


4

What are the commands to undo a commit?

To undo a commit you would use the following code:

$ git commit -m "undoing a commit"
$ git reset HEAD~1


5

One of your repositories is in a “detached HEAD” state. How do you fix this?

To return to the HEAD and no longer be in a detached HEAD state you would use the following code:

$ git checkout master


6

Explain the pros / cons of using Git for your research project.

Pros:

  • Keep a history of backup files
  • Easy to revert to a previous version if need be
  • Easy to collaborate and see who makes what changes
  • Access files remotely so you can work from anywhere

Cons:

  • Your work is public and can be accessed by anyone
  • Even if you're using a private repository it's still saved on a server
  • Not intuitive
  • Steep learning curve

7

Explain the pros / cons of hosting your project in a public (or private) repository on Github/Bitbucket/Gitlab/etc.

Pros of hosting your project in a public repository:

  • Easy to collaborate with others
  • Easily make your project accessible
  • Your project is backed up and can be accessed remotely

Cons of hosting your project in a public repository:

  • Anyone can access your project
  • Your data is no longer private

Pros of hosting your project in a private repository:

  • Your data is private
  • You can access your data remotely and back it up whilst remaining private

Cons of hosting your project in a private repository

  • Most private repositories are not free (i.e. GitHub)
  • Even though your repository is private, it is still located on a server somewhere..
